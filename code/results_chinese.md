## 背景介绍 — 这个模型是什么？为什么重要？

想象一条高速公路，卡车和乘用车共享车道。卡车行驶缓慢，会阻塞周围的车道。被卡在卡车后面的乘用车无法以正常速度行驶——它们被"困住"了。当足够多的车辆在缓慢的卡车后面堆积时，**交通堵塞**就形成了，并向上游蔓延，即使没有任何事故发生。这是高速公路上无明显原因拥堵的最常见原因之一。

这个模型——**多类别交通反应模型（Multiclass TRM）**——是一个数学框架，同时追踪三种车辆类型：

| 类别 | 符号 | 含义 | PCE权重（乘客车当量） |
|------|------|------|---------------------|
| 卡车 | **A** | 缓慢行驶、制造瓶颈的大型车辆 | 2.5（一辆卡车占据2.5辆乘用车的道路空间） |
| 自由乘用车 | **Bf** | 自由行驶的普通乘用车 | 1.0 |
| 被困乘用车 | **Bs** | 被堵在卡车后面无法正常行驶的乘用车 | 1.0 |

**m7版本**的核心创新是用一个简洁的公式——**概率阻断因子 P_block**——替代复杂的"车道换道"计算。这使模型更简单、更快速、数学上更清晰，同时保留了所有物理真实性。

## m7 版本升级的通俗解释

### 改了什么，为什么改

**旧模型（m3+m4版本）**：当一辆车被卡在卡车后面，模型需要检查*相邻车道*发生了什么——它们也被堵住了吗？车辆能否侧向逃脱？这需要在车道之间共享信息，使数学变得复杂，计算相互耦合。

**新模型（m7版本）**：我们只问一个问题：**"当前局部路段有多拥堵？"** 如果非常拥堵，车辆很可能被困住。如果道路畅通，车辆可以自由行驶。这个概率用一个公式捕捉：

$$P_{\text{block}}(x) = \left(\frac{\Omega_x}{\rho_{\max}}\right)^2$$

其中 $\Omega_x$ 是当前交通密度（路有多挤），$\rho_{\max}$ 是最大允许密度。结果始终在0到1之间：
- **空旷路段**（Ω = 0）→ P_block = 0 → 完全不阻断
- **堵死路段**（Ω = ρ_max）→ P_block = 1 → 最大阻断

这个单一数字驱动捕获率（自由车被困住的速度）和释放率（被困车逃脱的速度）。不需要车道间通信。

### 移除了什么

| 移除内容 | 原来是什么 |
|---------|-----------|
| 第3相侧向换道 | 整个计算步骤删除 |
| 参数：η_lat, ξ, R_gap, ω_sp, κ_hz | 5个控制车道间行为的参数 |
| HDF5数据集：gamma_left, gamma_right, E_trap | 旧版侧向逃逸数据不再存储 |

### 新增了什么

| 新增内容 | 含义 |
|---------|------|
| P_block(x) | 新的概率阻断公式 |
| 参数：η_block = 2.0 | 控制P_block上升速度的指数 |
| HDF5数据集：P_block(T, X, L) | 每个时间步、每个路段格子、每条车道的阻断概率均已存储 |

## 三相仿真循环

每个时间步（Δt = 0.5秒）依次执行三个阶段。可以将其理解为三个独立的"更新"操作，逐一进行：

```
─────────────────────────────────────────────────────────────────
第1相 — 谁被困住，谁被释放？（捕获与释放）
─────────────────────────────────────────────────────────────────
  输入：当前密度分布 f，当前占用密度 Ω

  1. 计算 P_block = (Ω / ρ_max)²              ← 有多拥堵？
  2. 计算 σ  = σ₀ · P_block · 暴露度 · B(Ω)   ← 捕获（困住）率
  3. 计算 μ  = μ₀ · exp(−P_block · θ_A / R_A) ← 释放率
  4. 用精确公式更新 f_Bf 和 f_Bs：
       f_Bf* = f_Bf · e^(−σΔt) + μ·F·Δt·φ(σΔt)
       f_Bs* = F − f_Bf*   （F = B类车辆总量，守恒）

  为什么用"精确"公式？因为σ在堵车时可以非常大（每秒数百万次）。
  普通Euler步骤会导致数值爆炸。精确矩阵指数法无论σ多大都给出正确答案。

─────────────────────────────────────────────────────────────────
第2相 — 车辆如何改变速度？（运动学）
─────────────────────────────────────────────────────────────────
  车辆在速度档之间加速（λ_acc）和减速（λ_dec）。
  关键规则：Bs（被困车）不能加速到速度阈值 i_thr 以上。
  这就是"运动学封锁"——被困车在物理上无法提速，因为前方有卡车。

  数学上是一个隐式三对角方程组（Thomas算法），因为速度变化率极端，
  显式方法会导致仿真崩溃。

─────────────────────────────────────────────────────────────────
第3相 — 车辆如何在空间中移动？（对流）
─────────────────────────────────────────────────────────────────
  车辆以其速度向前移动。Godunov限制器防止进入格子的车辆超过其容量：

       α = min(1, 可用空间 / 总需求)

  若 α < 1，所有车辆类别等比例缩减，
  没有任何类别获得优先权，也没有格子会溢出。
─────────────────────────────────────────────────────────────────
```

## 黎曼问题 — 两种基本交通波

"黎曼问题"是一个经典数学设置：两个具有不同交通状态的区域在边界相遇。会发生什么？在交通理论中恰好有两种可能的结果，本基准测试中**两者都经过了验证**。

### 什么是黎曼问题？（通俗解释）

想象一条高速公路，左半段车辆稀少（车速快），右半段车辆密集（车速慢）。在它们的交界处，可能发生两种情况：

1. **激波**：一道清晰的拥堵墙形成，并向稀疏区（上游）倒退——就像突然出现了一堵砖墙。后面的车辆突然需要刹车。

2. **稀疏波**：密集的交通区域逐渐向前扩展和平滑——就像波浪滚向大海。没有突兀的边界，只是逐渐疏散。

在本基准测试中，**两种波形由卡车瓶颈自然产生**。

---

### 黎曼问题 I — 激波（检验 V3-c）

**场景设置**：快速行驶的自由车（Bf车群）在格子 x = 59-69 注入。它们向右行驶，与静止在 x = 74-79 的卡车瓶颈发生碰撞。

**物理过程**：
- 卡车几乎不移动。后面的车辆堆积。
- 瓶颈处的密度 Ω 接近 ρ_max（道路满员）。
- P_block 跳升至1 → 车辆几乎立即被困住。
- B(Ω) → 无穷大 → 减速极端剧烈（奇异屏障激活）。
- 堆积向**上游**（后方）蔓延，就像红灯前的排队。

**如何验证（V3-c）**：
我们测量瓶颈*正上游*区域（格子 x = 68-73）的平均密度。如果激波向上传播，这个区域应该随时间变得更拥堵——即使那里本身没有发生任何事情。

- t = 0 时上游 Ω ≈ 0.0（初始为空）
- t = 50s 时上游 Ω 明显更高（激波已到达）
- 正向增量确认：**向上游传播的激波被检测到** ✓

---

### 黎曼问题 II — 稀疏波（检验 V3-g）

**场景设置**：瓶颈下游（格子 x = 80-110），道路初始畅通。

**物理过程**：
- 穿过瓶颈的车辆逃入空旷空间。
- Ω 立即低于 ρ_max → P_block 趋近0。
- μ（释放率）增大 → 被困的Bs车缓慢转变回Bf。
- 密度扇形**平滑向前扩展** — 没有尖锐边界。

**如何验证（V3-g）**：
我们在下游区域测量两件事：

1. **梯度平滑性**：相邻格子之间的密度变化应该很小。稀疏波没有突然的跳变——如果最大格子间密度差 < ρ_max的50%，说明是平滑的稀疏波而非新的激波。

2. **Bs丰度**：在自由流稀疏区，几乎不应有被困车辆。远下游（x = 90-110）的Bs密度应远小于瓶颈处（x = 74-79）的Bs密度。

两项指标均通过：**平滑稀疏波被确认** ✓

### V1 — 有效占用密度约束

![V1_occupancy](figures/V1_occupancy.png)

**什么是"占用密度" Ω？** 它是道路上所有车辆类别的加权总密度（每米道路的乘客车当量数）。一辆卡车算2.5辆乘用车。如果一辆卡车（PCE=2.5）和一辆自由车（PCE=1.0）占据同一米道路，则 Ω = 3.5/米。最大允许值为 ρ_max = 0.15 veh/m。

**左图 — [V1-a] 全局最大 Ω 随时间的变化：**

该图跟踪了整个道路网络在每个时刻所能找到的最高密度值。蓝色曲线振荡后逐渐下降。

- **t = 0 时**：Ω_max ≈ 0.145 — 这是格子74-79处的卡车瓶颈。卡车以0.058 veh/m的密度预先放置在那里。乘以PCE权重2.5：Ω = 2.5 × 0.058 = 0.145。这是最大容量的97%。
- 橙色注释箭头指向这一初始峰值，标注"初始Class A卡车瓶颈，Ω≈0.145"。
- 随着时间推移，卡车向下游扩散，Ω_max下降，在Bf车波到达并相互作用时产生振荡。
- **红色虚线**在0.15处是硬性限制 ρ_max。蓝色曲线在全部500个时间步内严格保持在其下方。**检验V1-a通过**：无超容量违规。

**右图 — [V1-c] t=0时各类对中间车道占用密度的贡献：**

该柱状图展示了仿真最开始时，每个路段格子中各车辆类别对总占用密度的贡献。

- **橙色柱（A类卡车）**：在格子74-79处有一个高峰，恰好达到0.145 — 确认卡车正确初始化，PCE权重（×2.5）正确应用。
- **蓝色柱（Bf类自由车）**：在格子59-69处有较宽的分布，约达0.06。这些是注入到瓶颈上游的乘用车。

**检验 V1-b 至 V1-e** 验证：密度永不为负；Ω公式计算精确（误差 < 10⁻¹⁰）；Bf和Bs的PCE权重相等（均为1.0），意味着互换一辆Bf和Bs不会改变道路占用密度。

**结论 — V1：5/5 全部通过。**

### V2 — 运动学转换率验证

![V2_kinematics](figures/V2_kinematics.png)

**背景**：车辆和卡车并非以单一速度行驶。每辆车在15个离散的"速度档"中分布（从慢到快）。每个时间步，车辆可以加速（移至更高档位）或减速（移至更低档位），由速率 λ_acc 和 λ_dec 控制。

**左上图 — [V2-a,g] 加速率 λ_acc 与占用密度 Ω 的关系：**

展示加速的概率如何取决于道路的拥堵程度。每条曲线代表一个车辆类别在一种速度下的情况。

- **实线** = A类卡车。**虚线** = Bf类。Bs（被困车）在高速档位的曲线为**平直零线** — 被困车在物理上无法加速到阈值速度以上。
- 所有曲线**从左向右递减**：当占用密度低（开阔道路）时，加速是可能的。随着Ω接近 ρ_max（右边缘，红色虚线），加速机会降为零。
- 这在物理上是直观的：当前方路段车满为患时，你无法加速。
- **检验V2-g**（Bs加速封锁）：Bs在 i ≥ i_thr 的曲线严格为零。这替代了旧版的侧向换道禁止——现在是纯粹的速度域规则。

**右上图 — [V2-c] 奇异屏障 B(Ω)：**

这是一个关键的安全机制。B(Ω)是一个乘数，随着密度接近最大值而使减速率急剧上升。图表使用**对数y轴**。

- **蓝色曲线（Bf类，η=2.0）**：随Ω接近0.15急剧上升，在瓶颈处达到约10³。
- **橙色曲线（A类，η=4.5）**：更陡。在Ω=0.145（橙色虚线）处，B_A ≈ 4.44×10⁶（红点标记）。这是正常减速的**四百余万倍**。
- 这种极端放大是导致仿真极端刚性的原因——也是为什么我们需要隐式求解器（第2相，Thomas算法）。
- **检验V2-c**：B(Ω=0.145) = 30^4.5 ≈ 4.155×10⁶。精确验证至6位有效数字。

**左下图 — [V2-d] t=0时空间中的减速率 λ_dec 分布：**

展示每个路段格子中各车辆类别被迫减速的强度，使用**对数y轴**。

- 三个类别在 x=74-79（红色阴影区，卡车瓶颈）均显示出巨大峰值。该处的值比开阔道路高出10³到10⁶倍。
- 这个峰值触发第1相的极端刚性：减速率如此之高，以致显式时间积分需要10⁻⁶秒或更小的时间步才能保持稳定。

**右下图 — [V2-e] 碰撞核不对称性：**

卡车（A）对后方车辆的"可见性"比其他乘用车更强。柱状图展示每个速度档对减速的贡献。

- **橙色柱（Bf跟随A）**：始终更高，确认 β × w = 0.06 × 2.5 = 0.15（卡车相互作用）。
- **蓝色柱（Bf跟随Bf）**：较低，β × w = 0.03 × 1.0 = 0.03（乘用车跟随乘用车）。
- 比率 = 5.0，与模型参数完全一致。

**结论 — V2：7/7 全部通过。**

### V3 — FVM空间对流 + 黎曼问题验证

![V3_fvm](figures/V3_fvm.png)

**背景**："FVM"代表有限体积法（Finite Volume Method）——流体力学中移动量穿越空间的标准技术。车辆以其速度向前移动，但我们必须保证格子中持有的车辆永远不超过其物理容量（ρ_max）。

**左上图 — [V3-a] 需求通量Ψ在下游满员时趋零：**

x轴是下游格子的占用密度。y轴是"需求通量"Ψ——想要进入的流量。每条彩色曲线对应不同的车速和密度组合。

- 所有曲线在左侧较高（下游空旷，有充足空间），随着下游Ω→ρ_max（红色虚线），所有曲线**都趋向零**。
- 这是供给过滤器：如果下一个格子已满，就不能再进入任何东西。无论多少辆车想向前移动，满员时Ψ=0。
- 全局Godunov限制器 α≤1 随后按比例缩减所有类别，确保总流入量与可用空间匹配。

**右上图 — [V3-c/g] Hovmöller时空密度图 — 黎曼问题的特征：**

这是V3中最重要的图。x轴是道路位置（格子0到150），y轴是时间（0到250秒），颜色表示平均密度（黄色=空旷，深红色=拥堵）。

- x=74-79处的**深红色垂直带**是卡车瓶颈。卡车从一开始就在那里制造高密度，贯穿大部分仿真时间。
- 仔细观察深色区域的**左边界**：随时间向左倾斜。这就是**向上游传播的激波** — 堵塞逐渐向后方蔓延。
- 瓶颈区域的**右侧**（x>80）：颜色更浅且逐渐消散。这是**向下游传播的稀疏波** — 车辆平滑地逃入空旷空间，没有尖锐边界。
- 两种黎曼波类型作为**几何图案在这张图中同时可见**。

**左下图 — [V3-f] 基本图 — 两种交通状态共存：**

每个点代表某个时刻某个路段格子的（密度，流量）状态。黑色虚线是理论最大值：q = v_max × ρ。

- **蓝色点簇（低密度，中低流量）**：自由流条件。车辆行驶快速，但数量不多。
- **红色/橙色散点（较高密度，低流量）**：拥堵条件。道路拥挤但车辆几乎不动。
- 双峰结构（两个独立的点簇）是移动瓶颈交通模型的典型特征。**检验V3-f通过**：B类密度标准差>0.005，确认两种不同交通状态共存。

**右下图 — [V3] t=0, 25秒, 50秒的三类密度空间快照：**

三个时刻的切片，显示每种车辆类别的集中位置。

- **橙色（A类卡车）**：在x=74-79处有尖锐峰值，几乎不移动——卡车行驶缓慢。
- **蓝色（Bf类自由车）**：从x=59-89处开始，向右传播的较宽波形。前端以注入速度（约30m/s）移动。
- **绿色（Bs类被困车）**：t=0时几乎为零（尚无被困），在x=74-79附近增长（Bf车被捕获），然后随瓶颈减弱而向下游消散。

**结论 — V3：7/7 全部通过**（含两个黎曼问题检验）。

### V4 — 概率阻断因子 P_block 验证（m7版本全新模块）

![V4_probability](figures/V4_probability.png)

**这个模块是全新的。** 它替代了旧版的侧向换道验证。目标是证明P_block——m7模型的核心——的行为完全符合理论要求。

**左上图 — [V4-c] P_block解析曲线与边界条件：**

平滑的上升蓝色曲线是 P_block(Ω) = (Ω/ρ_max)²。两个橙色散点标记端点：
- Ω=0时：P_block = 0（空旷道路无阻断）
- Ω=ρ_max时：P_block = 1（堵死密度时最大阻断）

### V6 — 刚性与算子分裂验证

![V6_stiffness](figures/V6_stiffness.png)

**背景**："刚性"是微分方程的一种数值性质，其中某些过程在截然不同的时间尺度上发生。这里，交通堵塞附近的减速（发生在微秒级别）必须与车辆在空间中移动（发生在0.5秒时间步级别）共存。如果用朴素方法以0.5秒步长模拟微秒级物理，数值会爆炸到无穷大。

**左上图 — [V6-a/b] 刚性比随时间的变化（对数坐标）：**

刚性比 S = 最快反应速率 / 最慢对流速率。S越大，系统越刚性，越需要特殊处理。

- 橙色曲线从 S≈1.57×10⁷ 开始——比对流速率高一千多万倍。这是"极端刚性"领域。

**中上图 — [V6-b] 时间尺度分离：**
