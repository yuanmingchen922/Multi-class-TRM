## 1. 模型概述

### 三类车辆

本模型将道路上的车辆分为三类，就像把马路上的交通参与者分成三组：

| 类别 | 名称 | 描述 | PCE权重 |
|------|------|------|---------|
| **Class A** | 卡车 | 重型货车，自由流速度上限 14 m/s = 50.4km/h| 2.5 |
| **Class Bf** | 自由乘用车 | 可以自由行驶的小汽车 | 1.0 |
| **Class Bs** | 被困乘用车 | 被卡车堵在后面、无法超越的小汽车 | 1.0 |

> **直觉理解**: 想象高速公路上，一辆慢速卡车前面堵了一排小汽车。前面的小汽车（Bf）可以换道绕过去，但紧跟在卡车后面的那几辆（Bs）被旁边的车道挡住了，暂时走不了。

### 核心物理量

- **f(m, i, x, l)**: 第 m 类、速度档 i、格子 x、车道 l 的车辆密度（单位：辆/米）
- **Ω(x, l)**: 有效 PCE 占用密度 = Σ w(m) × Σ f(m,i,x,l)，衡量道路"有多满"
- **ρ_max = 0.15 PCE/m**: 道路绝对容量上限，任何时候都不能超过

### 4 相算子分裂

模型每一步按照严格顺序执行 4 个物理过程：

```
Phase 1: 捕获/释放  →  Bf 遇到卡车被"困住"变 Bs，或 Bs 换道逃脱变 Bf
Phase 2: 速度动力学 →  车辆加速/减速（Thomas 隐式算法处理极端刚性）
Phase 3: 侧向换道   →  车辆在相邻车道间转移（Softplus 平滑驱动力）
Phase 4: 空间对流   →  车辆沿道路向前移动（有限体积法 + 全局 Godunov 限制器）
```

## 3. 数据生成逻辑

### 初始条件（病理性极端场景）

```
卡车瓶颈区 (格子 74–79):
  Class A 卡车密度 = 0.058 辆/米，速度 = 2 m/s（最低速）
  → 有效 PCE 占用 Ω = 2.5 × 0.058 = 0.145 ≈ ρ_max（几乎堵死）

Bf 高速注入区 (格子 60–70):
  Class Bf 乘用车密度 = 0.060 辆/米，速度 = 30 m/s（最高速）
  → 高速车群正在冲向堵车区，会触发大量"捕获"事件
```

这就像：高速公路上前方有一段慢速卡车堵路（Ω ≈ 0.145，接近极限），后方一群小车以最高速冲来，即将发生剧烈减速和捕获。

### 仿真流程（每步 Δt = 0.5 s）

每一步的核心计算如下：

**Phase 1 — 精确矩阵指数（捕获/释放）**

这是本模型最独特的部分。当 Bf 类乘用车遇到卡车时，被"捕获"成 Bs：

```python
# 逃逸门：邻道越堵，越逃不掉
G = (Ω_adj / ρ_max)^η_lat    # η_lat=2.0

# 困陷因子：左右都逃不掉才算真的被困
E = G_left × G_right

# 捕获率：卡车越多、越慢、越困，捕获率越高
σ = σ_0 × (1 + ξ·E) × S_tilde    # S_tilde = 卡车加权密度

# 精确积分（不用数值近似，直接解析解）
φ(z) = (1 - e^{-z}) / z          # z→0 时安全收敛到 1
f_Bf_new = f_Bf × e^{-S·Δt} + μ × (f_Bf+f_Bs) × φ(S·Δt)
```

**Phase 2 — 半隐式 Thomas 算法（速度动力学）**

由于瓶颈处刚性比高达 S ~ 10^7，普通显式方法会瞬间发散。Thomas 算法用三对角矩阵的精确解绕过这个问题：

```python
# 求解: (I - Δt·A) × f_new = f_old
# A 是三对角矩阵，表示速度档间的转移
# 前向消元 + 回代，计算量是 O(N)
```

**Phase 3 — Softplus 换道（侧向动力学）**

```python
# Softplus 把"如果密度差为正就换道"变成连续可微的函数
γ = κ × (1/ω) × ln[1 + exp(ω·y)] × gap_filter
# gap_filter: 目标车道越满，换道率越低
# Bs 的 γ 强制为 0（被困，禁止换道）
```

**Phase 4 — 全局 Godunov 通量限制器（空间对流）**

```python
# 聚合所有类的需求通量
D = (Δt/Δx) × Σ_m w(m) × Ψ(m)   # 总 PCE 需求

# 下游剩余容量
available = max(0, ρ_max - Ω_downstream)

# 全局缩放系数（要么全通，要么按比例限制）
α = min(1, available / D)

# 所有类按同一比例缩放，保证不超容量
Φ(m) = α × Ψ(m)
```

## 5. V2 — 动力学转换率 (`V2_kinematics.png`)

### 图表说明

**左上图：加速率 λ_acc 对比三类车辆**

这张图展示不同类型车辆在不同道路占用程度下的加速倾向。

- **横轴**：道路占用密度 Ω（0 = 空路，0.15 = 满载）
- **纵轴**：加速率（值越大，车辆越想加速）
- **实线 = Class A 卡车**：由于 η=4.5（陡峭），满载时加速率骤降更快
- **虚线 = Bf/Bs 乘用车**：由于 η=2.0，曲线较平缓
- **Bs 的虚线在 i≥i_thr 处被截为零**：这就是"加速封锁"——被困的车不能继续加速

> 通俗说：路越堵，越难加速。卡车对拥堵更敏感（η更大），被困乘用车(Bs)更是直接禁止超过某个速度。

**右上图：奇异屏障 B(Ω) 对数坐标图**

这是最能体现模型"防超载"机制的图。

- **纵轴（对数坐标）**：屏障值 B(Ω) = (ρ_max / max(ε, ρ_max-Ω))^η
- **Ω → ρ_max 时，B → ∞**：当道路接近满载，减速率趋近无穷大，物理上形成不可穿透的屏障
- **橙色竖线**：瓶颈处 Ω=0.145，此时 B_A = 30^4.5 ≈ 4.4×10^6（极端大！）
- **绿色横线**：期望计算值，两者完全吻合

> 通俗说：道路快堵满时，就好像一堵无形的墙挡在前面，车辆不得不急刹车，绝对不会"穿越"ρ_max。

**左下图：t=0 时 λ_dec 空间分布（对数坐标）**

展示道路各位置的减速率强度：

- **x=74–79（红色阴影区）**：卡车瓶颈区，减速率极高（图中峰值）
- **三类车**：Class A 的减速率最高（因为 β矩阵对卡车更大），Bf/Bs 次之

**右下图：β 矩阵非对称性验证**

验证"Bf 跟着 A 走比 Bf 跟着 Bf 走更容易减速"：

- **蓝柱（Bf跟Bf）** vs **橙柱（Bf跟A）**：橙柱是蓝柱的 5 倍
- 比率 = β[1,0]×w[0] / (β[1,1]×w[1]) = 0.06×2.5 / (0.03×1.0) = **5.0**
- 这符合直觉：被卡车挡住比被同类堵住更难超越

### 图表说明

这是验证换道模型数学合理性最直接的图：

## 9. V6 — 刚性与算子分裂 (`V6_stiffness.png`)

### 图表说明

"刚性"是数值计算中的概念：当方程中某些过程比另一些快**太多**时，就会出现数值不稳定。本节验证模型的极端刚性和解决方案。

**左上图：刚性比时序（对数坐标）**

- **刚性比 S** = 最大减速率 / 对流特征频率
- **初始 S ≈ 1.57×10⁷**（比阈值 10⁵ 大 100 倍！）
- **最大 S ≈ 2.2×10¹⁵**（极端数字，说明模型的刚性极其严重）

> 通俗说：方程里最快的过程（减速反应）比最慢的过程（车辆前进）快 **10亿倍以上**。如果用普通数值方法，时间步长需要缩小到 10⁻¹⁵ 秒才能稳定，完全不可用。

**右上图：时间尺度对比**

- **橙线（τ_relax）**：减速反应的特征时间，极短（纳秒量级）
- **蓝横线（τ_adv）**：对流特征时间 = Δx/v_max = 0.667 s
- **灰横线（Δt）**：实际时间步长 0.5 s
- 三者相差 15 个数量级 → 这就是为什么必须用隐式方法

**右上图：显式 vs 隐式一步结果（Class A）**

最直观的对比图：

- **灰柱（初始状态）**：卡车在最低速度档集中
- **蓝柱（Thomas 隐式）**：一步之后仍然物理合理，密度分布平滑
- **橙柱（显式 Euler）**：一步之后部分速度档出现负值或超出范围（发散的开始！）

**左下图：显式 Euler 5 步发散轨迹**

逐步展示显式方法如何失控：
